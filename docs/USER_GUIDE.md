## 1. 시작하기

### 설치

[Releases](https://github.com/HyunjoonKwak/acut/releases)에서 `.dmg`를 받아 응용 프로그램에 넣습니다.
ad-hoc 서명이라 "손상됨" 경고가 뜨면 터미널에서 한 번만:

```bash
xattr -cr /Applications/에이컷.app
```

> **스마트 폴더에서 업그레이드한 경우** — 첫 실행 때 기존 라이브러리(사진 DB, 설정, 썸네일)가 자동으로 이전됩니다. 별도 작업이 필요 없습니다.

### 화면 구성

좌측 **아이콘 레일**이 유일한 최상위 내비게이션입니다. 위에서 아래 순서가 곧 사용 순서입니다:

| 레일 | 하는 일 |
|------|---------|
| 🗂 **작업대** | 전 과정 지휘 — 파이프라인 현황, 장치 가져오기, 스마트 정리 |
| 🖼 **라이브러리** | 사진 탐색, 태그·앨범·코멘트, 지도 |
| ✅ **고르기** | 중복·연사·수동 리뷰로 남길 사진 선별 |
| 📁 **정리** | 자동 분류, 폴더 작업, 동기화, 히스토리 |
| ☁️ **NAS** | Synology NAS 업로드 |
| ⚙️ 설정 | 테마·언어·감시·스케줄·MCP |

상단 바에는 검색창(파일명·경로·카메라·**코멘트** 검색)과 스캔 진행률, 라이브러리 통계만 있습니다.

### 첫 사진 가져오기

세 가지 방법이 있습니다:

1. **작업대 → 가져오기 스테이션 → "폴더 추가"**
2. **SD카드/외장 드라이브 연결** → 작업대 상단에 알림이 뜨면 **"작업대에 올리기"** 클릭 (등록+스캔 원클릭)
3. **라이브러리 사이드바 → 장치 패널** → 장치별 "전체 검색" 또는 "폴더 선택"

스캔은 2단계로 진행됩니다: 빠른 파일 목록(즉시) → EXIF·썸네일 처리(백그라운드, 상단 진행률 표시).

> 내장 디스크(Macintosh HD) 전체를 검색해도 됩니다 — Library, System, 앱 번들 등 시스템 경로는 자동으로 건너뜁니다.

## 2. 작업대

앱을 켜면 처음 만나는 화면이자, 일을 시작하는 곳입니다.

### 작업 대상 선택

우측 상단 드롭다운에서 **전체 라이브러리** 또는 특정 소스 폴더를 고르면, 아래 파이프라인의 모든 수치(장수·중복·미업로드 등)가 그 범위 기준으로 다시 계산됩니다 — "지금 다룰 재료"를 정하는 단계입니다.

### 파이프라인 보드

**01 가져오기 → 02 고르기 → 03 정리 → 04 올리기** 4개 스테이션이 각자 현황을 보여줍니다:

- **가져오기**: 소스 수 · 전체 장수 · 용량 — [폴더 추가] [다시 스캔]
- **고르기**: 중복 그룹 · 연사 그룹 수 — [고르러 가기]
- **정리**: 자동 분류·폴더 바로가기
- **올리기**: 미업로드 장수 · NAS 연결 상태 — [NAS 업로드]

버튼을 누르면 해당 작업 공간으로 바로 이동합니다.

### 스마트 정리

**중복 → 연사 → 폴더 정리**를 순서대로 안내하는 통합 위저드입니다. 단계마다 미리보기를 보여주고, 원치 않는 단계는 건너뛸 수 있습니다.

### 최근 작업 · 자동화

- 최근 작업 3건이 표시되며 **"되돌리기"** 한 번으로 복원됩니다
- 폴더 감시 / 스케줄 / MCP 서버의 동작 상태를 한눈에 확인합니다

## 3. 라이브러리

### 보기 방식

상단 탭으로 전환합니다: **격자 / 리스트 / 지도**

- **격자**: 썸네일 그리드. 우측 슬라이더 또는 `⌘+`/`⌘−`로 썸네일 크기 조절(80~320px, 설정에 저장됨)
- **리스트**: 정렬 가능한 표(파일명·폴더·촬영일·수정일·크기·해상도·유형), 열 너비 조절
- **지도**: GPS 좌표가 있는 사진을 클러스터로 표시. 클러스터 클릭 → 해당 위치 사진 필터

### 사이드바 (필터)

- **소스**: 등록된 폴더 목록 — 호버 시 다시 스캔/삭제
- **라이브러리**: 전체 / 사진만 / 영상만, 날짜별·폴더별 그룹
- **태그 / 앨범**: 항목을 클릭하면 갤러리가 즉시 필터링됩니다. 다시 클릭하면 해제. "관리"로 태그·앨범 관리 화면 이동
- **장치**: 내장·외장 저장장치 목록 — 전체 검색 / 폴더 선택 / (외장) 안전 추출

활성 필터는 상단에 칩(`# 가족 ✕`)으로 표시되고 ✕로 해제합니다.

### 인스펙터

사진을 클릭하면 우측에 인스펙터가 열립니다:

- **정보**: 크기·해상도·촬영일·형식·폴더
- **태그**: 칩으로 표시, 입력창에서 추가(기존 태그 자동완성)·✕로 제거
- **앨범**: 소속 앨범 칩 표시·제거, 드롭다운으로 담기
- **코멘트**: 메모를 남기면 그리드에 💬 배지가 붙고, **상단 검색창에서 코멘트 내용으로도 검색**됩니다 (예: "인화"라고 메모해두면 나중에 "인화"로 검색)

### 배지

| 배지 | 의미 |
|------|------|
| ✓ (초록) | NAS 업로드 완료 |
| 💬 (노랑) | 코멘트 있음 |
| ▶ | 영상 |

## 4. 고르기

세 가지 모드가 탭으로 통합되어 있고, 탭에 남은 그룹 수가 표시됩니다.

### 중복

3단계 해시(크기 → 퀵해시 → SHA-256)로 완전 동일 파일을 찾습니다.

1. **중복 스캔** 실행
2. 그룹별로 **보관 파일 지정** (기본: 첫 파일)
3. **삭제 미리보기**로 영향 확인 → 그룹 단위 또는 전체 휴지통 이동
4. 잘못 잡힌 그룹은 **무시(S)**

### 연사

몰아 찍은 사진(기본 5초 간격) 중 베스트만 남깁니다.

1. **탐지 시작** — 선명도(라플라시안)·노출(히스토그램) 기반 품질 점수 계산
2. 그룹마다 베스트가 자동 선정됨 — 마음에 안 들면 `1`~`9`로 다른 사진 지정
3. **삭제 미리보기** → 베스트 외 사진 휴지통 이동

### 한 장씩

폴더를 골라 전체를 한 장씩 훑으며 결정합니다.

- `Space` = B컷 마킹 후 다음, `K` = 유지 후 다음
- 품질 점수 오버레이와 저품질(40% 미만) 경고 표시
- **자동 분석** 토글: 연사 비베스트·저품질 사진을 미리 마킹해줌
- 마지막에 **마킹된 파일 일괄 휴지통 이동**

> 모든 삭제는 **미리보기 → macOS 휴지통** 경유입니다. 영구 삭제는 없습니다.

## 5. 정리

### 자동 분류

- **날짜별**: EXIF 촬영일 기준 `YYYY-MM-DD` 폴더로
- **유형별**: 사진/영상 분리
- 실행 전 **Before-After 미리보기**, 실행 후 히스토리에서 되돌리기 가능

### 폴더

파일 탐색기 + 폴더 작업 도구입니다.

- **듀얼 패널**: 좌측 트리에서 폴더를 클릭하면 우측에 그 폴더의 내용이 나타납니다. 더블클릭으로 폴더 진입(`..`으로 상위), 파일 더블클릭은 Quick Look
- **탐색**: 트리 구조(5단계), 검색·유형 필터, 용량 통계
- **선택**: 파일은 클릭, **폴더는 ⌘클릭**으로 선택 (여러 개 가능)
- **툴바**: 선택 항목에 대해 —
  - 📋 **복사**: 폴더 구조 통째로 복사 (대상 폴더 선택)
  - 📁 **이동**: 다른 볼륨으로도 이동 가능, 히스토리에 기록됨
  - ✏️ **이름 변경**: 1개 선택 시
  - 🗑 **휴지통**: 폴더는 내용까지 함께
- **드래그&드롭**: 파일·폴더를 트리의 폴더 위로 끌면 이동, `⌥` 누르면 복사
- **날짜 폴더 정리**: `20260614` 형식 폴더를 `2026-06-14`로 일괄 변환(병합 옵션)

### 동기화

소스 → 대상 단방향 미러링. SD카드 백업 등에 사용합니다.

1. 소스/대상 선택, 제외 패턴(glob) 입력
2. **미리보기(Dry-Run)**: 신규/업데이트/충돌/고아 파일 집계
3. 충돌은 개별 또는 일괄로 **덮어쓰기 / 이름 변경 후 복사 / 건너뛰기**
4. 자주 쓰는 설정은 **프리셋**으로 저장

### 히스토리

모든 이동·정리·폴더 작업이 배치로 기록됩니다. **되돌리기**를 누르면 역순으로 복원되고 라이브러리 경로도 함께 맞춰집니다.

## 6. NAS 업로드

Synology DSM에 직접 연결해 업로드합니다 (별도 클라이언트 불필요).

1. **연결**: 주소(QuickConnect 제외, IP/도메인:포트) · 계정 · 비밀번호 — HTTPS 자체 서명 인증서면 "TLS 검증 끄기"
2. **대상 폴더**: 원격 브라우저에서 선택하거나 새로 생성
3. **옵션**:
   - **촬영일 날짜 폴더 정리**: NAS에 `YYYY-MM-DD` 구조로
   - **B컷 제외**: 마킹된 B컷을 빼고 A컷만
   - **업로드한 항목 건너뛰기**: SHA-256 원장 기반 — 재실행해도 중복 업로드 없음
   - 동일 이름 덮어쓰기
4. **범위**: 전체 라이브러리 또는 갤러리에서 선택한 항목만
5. 진행률(성공/건너뜀/실패)이 표시되고 언제든 취소할 수 있습니다. 완료된 사진은 갤러리에 ✓ 배지

## 7. 설정 & 자동화

⚙️ 설정은 4개 탭입니다:

| 탭 | 항목 |
|----|------|
| 일반 | 테마(라이트/다크/시스템), 언어(한국어/English), 설정 초기화 |
| 감시 | 파일 변경 자동 감지 ON/OFF, 감지 간격(기본 500ms), 변경 시 자동 스캔 |
| 스케줄 | Cron 반복 작업 — 예: `0 0 * * *`(매일 자정) 스캔/정리/동기화 |
| MCP | AI 에이전트 제어 서버 시작/중지 — 도구: scan, get_stats, detect_duplicates, organize, sync, get_media_list |

## 9. 자주 묻는 질문

**Q. 앱이 "손상되었기 때문에 열 수 없습니다"라고 나옵니다.**
ad-hoc 서명 때문입니다. `xattr -cr /Applications/에이컷.app` 실행 후 다시 여세요.

**Q. 스마트 폴더 시절 사진들이 그대로 있나요?**
네. 첫 실행 때 라이브러리 전체(DB·설정·썸네일)가 새 위치로 자동 이전됩니다.

**Q. 삭제한 사진을 복구하려면?**
모든 삭제는 macOS 휴지통으로 이동합니다. 휴지통에서 "되돌려 놓기" 하세요. 이동·정리 작업은 앱 내 **히스토리**에서 되돌립니다.

**Q. 내장 디스크 전체를 검색해도 되나요?**
됩니다. 시스템 경로(Library, System, 앱 번들, node_modules 등)는 자동 제외되어 실사용 사진만 수집됩니다.

**Q. 데이터는 어디에 저장되나요?**
`~/Library/Application Support/com.acut.media/` — `smart_category.db`(라이브러리), `config.yaml`(설정), `thumbnails/`(썸네일)

**Q. 원본 파일이 수정되나요?**
스캔·고르기·태그·코멘트는 원본을 건드리지 않습니다. 파일이 움직이는 것은 정리(이동)·동기화(복사)·휴지통 이동뿐이며, 전부 미리보기와 되돌리기를 제공합니다.
